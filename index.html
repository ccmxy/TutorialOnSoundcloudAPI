So, what is an API, anyway?

API stands for Application Protocol Interface. To put it simply, when you're talking about an API, you're talking about a set of tools that you can use to request different services that can help you create software or web applications. A metaphor based in real-life might be operating a car. You know that behind the dashobard are a set of tools that you can use to request different services from your car, like "turn on blinker" or "switch to a higher gear." You don't need to know the ins-and-outs of how a car works to use these tools, you just need to know how to use them.

Today, we're going to do something very simple: We're going to use the Soundcloud API to add a simple widget to an HTML page. Since I understand that you may not have any experience with web development, we're going to start from scratch.

<!DOCTYPE html> <html> <head> <meta charset="UTF-8"> <title>SoundCloudAuth</title> </head> <body> <h3>Sound Cloud Api Example</h3> </body> </html>

Getting our credentials.

The SoundCloud API uses an authorization framework called OAuth 2.0. What this means is that it, from an application (such as the web application we're making) that's registered with SoundCloud may allow users to log into their account without asking for the user's credentials.

Register your app

The first thing we'll need to do is register our application. Go to http://soundcloud.com/you/apps to register your application. You will need to either log into or create a SoundCloud account in order for this to work.

From here, click "register application." You will be taken to this page:

Choose a name and check the Developer Policies box. We'll call ours "SoundCloudAPIExample."

Now comes the tricky part.

At this point, you need to have a url that your html file can be reached at. If you don't know how to do this, try signing up for an Amazon Web Services account and use the S3 Bucket feature to get a URL that your file index.html can be accessed from.

Getting the site online

That's the first URL that we need to register our app. Let's enter it now.

Now, we still need one more thing to register our app so that we can authenticate users: That's the redirect uri.

First, go into the folder on your computer that our index.html is saved in, and add in one more file: call this one callback.html. In callback.html, paste this code:

<!DOCTYPE html> <html> <head> <meta charset="utf-8"> <title>Callback</title> </head> <body onload="window.setTimeout(opener.SC.connectCallback, 1);"> <p> This page should close soon </p> </body> </html> Now, save the file and upload callback.html to the same bucket as you did before, and don't forget to also make this file public. Just like you did with index.html, right click and hit "properties" to grab the url. It should be the same as the last url but with index.html replaced with callback.html.
Now paste that url into your app at soundcloud.you.apps and hit "save".

Finally, we're going to add the javascript to our index file that will propt the user to connect soundcloud to our web app. Add this javascript to index.html just before the end of the , replacing YOUR_CLIENT_ID and http://example.com/callback with the proper credentials: <script src="https://connect.soundcloud.com/sdk/sdk-3.0.0.js"></script> <script> SC.initialize({ client_id: 'YOUR_CLIENT_ID', redirect_uri: 'http://example.com/callback' }); // initiate auth popup SC.connect().then(function() { return SC.get('/me'); }).then(function(me) { alert('Hello, ' + me.username); }); </script>
